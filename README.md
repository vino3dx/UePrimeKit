# UePrimeKit

基于 **Unreal Engine 5.1** 开发的 C++ 基础项目及核心插件包。本项目主要用于开发专为虚幻引擎设计的 C++ 插件与工具，并通过反射机制将其暴露给蓝图，作为高效的蓝图节点使用。

---

## 🛠️ 环境要求

在克隆和运行本项目前，请确保您的开发环境满足以下条件：
* **引擎版本**：Unreal Engine 5.1.x
* **IDE**：Visual Studio 2022 (需安装 "使用 C++ 的游戏开发" 组件以及 "Unreal Engine 侦测工具")
* **系统环境**：Windows 10 / 11 (64-bit)

---

## 🚀 首次下载使用步骤（项目构造指南）

由于 Git 仓库中忽略了所有本地编译的二进制文件（`Binaries`、`Intermediate` 等），**直接双击 `.uproject` 文件会提示缺失模块而无法打开**。请务必按照以下步骤进行项目构造：

### 1. 克隆项目
将项目克隆到本地非中文路径下：
```bash
git clone <你的GitHub仓库地址>

```

### 2. 生成 Visual Studio 项目文件 (Generate Files)

1. 进入项目根目录。
2. 鼠标右键点击 `UePrimeKit.uproject` 文件。
3. 在右键菜单中选择 **"Generate Visual Studio project files"**。
4. 稍等片刻，根目录下会自动生成 `UePrimeKit.sln` 解决方案文件。

### 3. 初次编译项目 (Build)

1. 双击打开 `UePrimeKit.sln` 进入 Visual Studio。
2. 确保顶部工具栏的 **解决方案配置** 选择为 `Development Editor`，**解决方案平台** 选择为 `Win64`。
3. 在右侧资源管理器中右键点击 `UePrimeKit` 项目，选择 **生成 (Build)** 或 **重新生成 (Rebuild)**。
4. 编译成功后，即可双击 `UePrimeKit.uproject` 正常拉起虚幻引擎编辑器；或者直接在 VS 中按 `F5` 启动调试。

## 📦 包含的自定义插件 (Plugins)

项目内嵌的自定义插件均存放在 `/Plugins` 目录下，并已做好多版本兼容设计：

1. **Super Tool** (版本 4.5.0)
* **功能**：多功能通用工具插件。支持打包后的 UDP 消息收发、外部 file 加载（TXT、图片、INI）运行时动态加载，以及运行时窗口位置与大小的调整。


2. **LicenseManager** (版本 1.0.0)
* **功能**：UE5 运行时许可证管理器。为离线授权系统提供 AES-256-CBC 加密许可证文件验证、Base64 解码、JSON 解析和过期日期检查功能。

## ⚠️ 核心开发注意事项

### 1. 严格的代码命名规范 (`.editorconfig`)

项目根目录和插件目录已配置好 `.editorconfig` 文件，用于规范虚幻引擎的 C++ 命名约定：

* 请确保 Visual Studio 中已启用 **Unreal Engine 命名约定检查器**。
* 编写暴露给蓝图的类和函数时，请遵循 UE 官方前缀规范（如 `UObject` 派生类加 `U`，`AActor` 派生类加 `A`，结构体加 `F`）。

### 2. C++ 暴露给蓝图的修改规则 (Live Coding)

项目默认采用 **Live Coding (连坐编码)** 机制以提高开发效率：

* **仅修改 `.cpp` 逻辑**：在 VS 中修改后，可保持引擎开启，直接按 **`Ctrl + Alt + F11`** 触发闪电编译，修改将立即可见。
* **修改 `.h` 头文件**（如新增 `UPROPERTY`、`UFUNCTION` 蓝图节点）：由于涉及到虚幻反射系统的改变，**强烈建议先关闭 UE 编辑器**，在 VS 中修改后按 `F5` 重新编译并拉起引擎，以防蓝图缓存或内存冲突。

### 3. 多版本向下兼容策略

本项目基于 **UE 5.1** 建立，目的是为了保持最好的向下兼容性：

* 在当前低版本中开发的 C++ 插件和蓝图资产可以无缝复制、升级到更高版本（如 UE 5.4）的项目中使用。
* 请勿直接将高版本（5.4+）的 `.uasset` 资产（如蓝图、材质）直接扔进本项目，高版本资产无法在低版本引擎中反序列化。
